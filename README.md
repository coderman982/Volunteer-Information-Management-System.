# Volunteer Information Management System (VIMS)

A lightweight web application built for coordinating volunteer activities, tracking event registration, and managing service hours. VIMS provides separate dashboards for administrators and volunteers, ensuring smooth operations for non-profit and community organizations.

---

## 🚀 Getting Started

Follow these steps to set up and run the project locally.

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed (version 16.x or higher is recommended).

### Installation
1. Clone the repository or extract the project source files.
2. Navigate to the project root directory.
3. Install the dependencies:
   ```bash
   npm install
   ```

### Running the Application
1. Start the server:
   ```bash
   npm start
   ```
2. Open your web browser and navigate to:
   ```
   http://localhost:3000
   ```

## ✨ Key Features

VIMS uses **Role-Based Access Control (RBAC)** to offer customized views and actions depending on the logged-in user:

### 👤 Volunteer Dashboard
* **Profile Management:** Edit and save volunteering details, including:
  * Key skills (e.g., Gardening, Teaching, First Aid).
  * Area of interest (e.g., Environment, Technology, Education).
  * Personal availability (e.g., Weekends, Weekdays, Flexible).
* **Event Participation:**
  * Browse the list of upcoming and completed events.
  * Register for upcoming events with a single click.
  * Cancel event registration if availability changes.
* **Hours Logging:**
  * Log hours served for completed events.
  * Add descriptions detailing contributions made.
  * Track approval status (Pending ⏳, Approved ✅, Rejected ❌) from the admin panel.

### 🔑 Administrator Portal
* **Real-time Statistics:** Access the Admin Dashboard displaying key metrics:
  * Total registered volunteers.
  * Total planned events.
  * Total approved service hours.
  * Number of pending hour logs requiring review.
  * Leaderboard of top volunteers (sorted by approved hours).
  * Registration status across all events.
* **Event Management:**
  * Create new volunteer events (specify title, description, date, and location).
  * Edit event details or update event status (e.g., change from upcoming to completed).
  * Delete events (which cascadingly removes all associated registrations and log history).
* **Volunteer List Overview:**
  * View details of all registered volunteers, including their skills, availability, and cumulative approved service hours.
* **Hours Log Approvals:**
  * Review all logged hours submitted by volunteers.
  * Approve or reject logs, instantly updating the volunteer's total hours.

## 🛠️ Technology Stack

* **Backend:** Node.js, Express.js (REST API development)
* **Database:** SQLite3 (Lightweight, serverless relational database)
* **Security & Auth:** JSON Web Tokens (JWT) for session control, `bcryptjs` for password hashing, and cookies for secure authentication.
* **Frontend:** Single Page Application (SPA) built with Vanilla HTML5, CSS3, and JavaScript (ES6+).
